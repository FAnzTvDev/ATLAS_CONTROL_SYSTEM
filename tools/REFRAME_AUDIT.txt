EXECUTIVE SUMMARY
================================================================================

The ATLAS codebase contains a MATURE, MULTI-LAYERED end-frame reframing system
used for:
  1. Master-chain continuity (end-frame → next shot's first-frame)
  2. Multi-angle variants (1 master frame → 3 reframed angles via nano-pro-edit)
  3. Continuity memory-driven reframe candidate selection
  4. End-frame validation gates (post-video analysis)

FINDING: The system is DEEPLY INTEGRATED across 8 production files.
         Disabling it would require changes to orchestrator_server.py,
         atlas_v26_controller.py, master_shot_chain_agent.py, and others.

KEY COMPONENTS ACTIVE:
  ✅ End-frame extraction (post-video, per-shot)
  ✅ Reframe validation gates (analyzing extracted frames)
  ✅ Nano-banana-pro/edit model calls (fal-ai/nano-banana-pro/edit)
  ✅ First-frame-variants directory (stores angle reframes)
  ✅ Multi-candidate generation (num_outputs/num_images = 3)
  ✅ Continuity memory reframe candidate selection
  ✅ Master-chain system (1 master + N reframes)

ARCHITECTURE OVERVIEW
================================================================================

FIVE DISTINCT REFRAMING WORKFLOWS:

1. MASTER-CHAIN SYSTEM (atlas_agents/master_shot_chain_agent.py)
   ├─ Generates 1 master frame (wide/establishing)
   ├─ Reframes master into 1 angle variant per shot (via nano-pro-edit)
   ├─ Cost: 1 master + (N shots × 1 reframe) = 1 + N calls
   ├─ Output: first_frame_variants/{shot_id}_{angle_name}.jpg
   └─ FIRES: master-chain agent workflow (optional, user-selected)

2. MULTI-CANDIDATE GENERATION (orchestrator_server.py lines 22258-22267)
   ├─ Generates 3 candidate frames per hero shot (num_outputs=3)
   ├─ Scored by keyframe gate, best selected, others saved as variants
   ├─ Cost: N shots × 3 candidates = 3N calls
   ├─ Output: first_frames/{shot_id}.jpg (best) + variants/ (alternates)
   └─ FIRES: generate-first-frames (Phase D.2 — multi-candidate loop)

3. END-FRAME REFRAMING (orchestrator_server.py lines 8730-8840)
   ├─ After video generation, extracts last frame
   ├─ Reframes end-frame into 3 angle variants via nano-pro-edit
   ├─ Angle data: {name: "reverse", "wide", "medium_close"}
   ├─ Cost: 1 video → 3 reframes = 3 calls per shot
   ├─ Output: first_frame_variants/{shot_id}_{angle_name}.jpg
   └─ FIRES: render-scene (after LTX/Kling video completion)

4. CONTINUITY MEMORY REFRAMING (tools/continuity_memory.py + orchestrator_server.py)
   ├─ Analyzes previous shot end-frame + spatial state
   ├─ Generates reframe candidates (5 strategies)
   ├─ Orchestrator selects best candidate for next shot
   ├─ Cost: Analysis only (no generation cost, informs strategy)
   ├─ Output: Influence on next shot's prompt via continuity delta
   └─ FIRES: generate-first-frames (Phase E1 — blocking analysis)

5. CREATION PACK MULTI-ANGLE REFS (tools/creation_pack_validator.py)
   ├─ Character ref packs: headshot (P1) + 3_quarter (P2) + full_body (P3)
   ├─ Location ref packs: wide (P1) + reverse_angle (P2) + medium_interior (P3)
   ├─ Each angle = IMAGE-TO-IMAGE reframe from P1 master
   ├─ Cost: Auto-recast generates missing refs on demand
   ├─ Output: character_library_locked/{char}_{angle}.jpg + location_masters/
   └─ FIRES: creation-pack validation (Phase 6 of pre-generation checklist)
