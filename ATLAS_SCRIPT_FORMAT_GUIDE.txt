ATLAS V21.9 — SCRIPT INGESTION FORMAT GUIDE
=============================================
How to write screenplays that produce AAA output from the ATLAS pipeline.

This guide was reverse-engineered from the parser (orchestrator_server.py lines 31324-32022),
the fix-v16 enrichment pipeline, and the 50-point quality audit system.

=== OVERVIEW ===

The ATLAS pipeline processes screenplays in 9 stages:
1. SCRIPT IMPORT — Regex parser extracts scenes, characters, dialogue, action lines
2. STORY BIBLE — LLM expands narrative beats, adds visual detail
3. FIX-V16 — Camera defaults, gold standard, ABC coverage, beat fidelity, segments
4. AUTO-CAST — Matches characters to AI actors from library (with reference images)
5. ASSET CREATION — Location masters + character headshots assigned as reference images
6. GENERATE FIRST FRAMES — nano-banana-pro creates images from prompts + reference images
7. MASTER CHAIN — End-frame reframing creates angle variants, chains shots together
8. GENERATE VIDEOS — LTX-2 animates frames into video clips
9. STITCH — FFmpeg concat into final output

The script format directly impacts steps 1-3. Bad formatting = bad shots = endless manual fixes.

=== FORMAT RULES ===

1. SCENE HEADERS
   Format: INT. LOCATION - TIME
   or:     EXT. LOCATION - TIME
   or:     INT/EXT. LOCATION - TIME

   Good:   INT. RAVENCROFT MANOR LIBRARY - NIGHT
   Good:   EXT. COASTAL ROAD - DAY
   Good:   INT/EXT. VILLAGE PUB - EVENING

   Bad:    **INT. RAVENCROFT MANOR LIBRARY - NIGHT**    (markdown bold)
   Bad:    ### SCENE 7 - THE LIBRARY                    (markdown header)
   Bad:    INT RAVENCROFT MANOR LIBRARY - NIGHT          (missing period after INT)

   RULE: Scene header MUST start with INT. or EXT. or INT/EXT.
   RULE: Use hyphen (-) or en-dash (–) to separate LOCATION from TIME
   RULE: NO markdown formatting anywhere (**bold**, ### headers, --- dividers)
   RULE: Time of day should be: DAY, NIGHT, MORNING, EVENING, DAWN, DUSK, CONTINUOUS

2. CHARACTER NAMES
   Format: CHARACTER NAME (all caps, standalone line)
   Optional: CHARACTER NAME (V.O.) for voice-over
             CHARACTER NAME (O.S.) for off-screen
             CHARACTER NAME (CONT'D) for continued dialogue

   Good:   EVELYN
   Good:   LADY MARGARET
   Good:   CHILD (V.O.)
   Good:   DR. WARD

   Bad:    Evelyn                          (not all caps)
   Bad:    EVELYN RAVENCROFT (V.O.)        (too long — parser has 40 char limit)
   Bad:    LADY MARGARET RAVENCROFT        (use shorter form after first intro)

   RULE: Character name MUST be ALL CAPS
   RULE: Character name MUST be on its own line (nothing else on that line)
   RULE: Max 40 characters including (V.O.) etc.
   RULE: Use SHORT form after first appearance (MARGARET not LADY MARGARET RAVENCROFT)

   NAMING CONVENTION FOR PIPELINE:
   - First appearance: Full name in action line (EVELYN RAVENCROFT, early 30s, sharp eyes)
   - Dialogue cue: Short form (EVELYN, MARGARET, ARTHUR, DR. WARD)
   - The parser tracks ALL forms and creates alias mappings

3. DIALOGUE
   Format: Indented text under character name

   Good:   EVELYN
           I don't know any Lady Margaret.

   Bad:    EVELYN
           I don't know any Lady Margaret.
           She left when I was a child.
           I barely remember her face.
           (Too many lines — parser takes them ALL as dialogue)

   RULE: Keep dialogue to 1-2 lines per speaking turn
   RULE: Long speeches should be broken into multiple turns with action between
   RULE: Pipeline has 250-char limit for LTX dialogue injection

4. PARENTHETICALS
   Format: (direction) on its own line between character name and dialogue

   Good:   MARGARET
           (whispering)
           The binding holds.

   RULE: Always on own line, always in parentheses
   RULE: These become acting direction in the prompt

5. ACTION LINES
   These are THE MOST IMPORTANT lines for the pipeline.
   Every action line becomes a visual beat. The parser classifies them as:
   - "action" — character blocking, physical movement
   - "atmosphere" — lighting, weather, mood keywords
   - "broll_direction" — camera direction, visual cues

   Good:   Evelyn steps through the doorway into the foyer.
   Good:   Candlelight flickers across the stone walls. Shadows pool in corners.
   Good:   She picks up the leather journal from the desk.

   Bad:    (nothing — empty gaps between dialogue lose visual beats)
   Bad:    It was a dark and stormy night. (pure prose, no visual beat)

   RULE: Every action line should be ONE shootable moment
   RULE: Name characters in action lines (parser extracts characters_present)
   RULE: Include physical verbs: walks, picks up, turns, kneels, runs
   RULE: Include props: journal, candle, portrait, key, letter
   RULE: Include atmosphere: candlelight, shadows, rain, wind, dust

6. TRANSITIONS
   Format: TRANSITION TYPE: on its own line

   Good:   CUT TO:
   Good:   SMASH CUT TO:
   Good:   DISSOLVE TO:
   Good:   FADE IN:
   Good:   FADE OUT:

   RULE: Parser recognizes: CUT TO, DISSOLVE TO, FADE IN, FADE OUT,
         FADE TO, SMASH CUT, MATCH CUT, JUMP CUT, WIPE TO, TIME CUT

7. INTERCUT SCENES
   Format:
   INT. FIRST LOCATION - TIME
   (dialogue/action)
   INTERCUT WITH:
   INT. SECOND LOCATION - TIME
   (dialogue alternates between locations)

   The parser tracks which character belongs to which location.
   RULE: Use INTERCUT WITH: on its own line
   RULE: Follow immediately with the second location's scene header
   RULE: Characters introduced BEFORE INTERCUT are at the first location
   RULE: Characters introduced AFTER INTERCUT are at the second location

8. VOICE-OVER CHARACTERS
   For characters heard but NEVER seen (like CHILD in Ravencroft):
   - Always use (V.O.) suffix: CHILD (V.O.)
   - NEVER put V.O. characters in action lines as visible
   - Pipeline tags these as _child_vo=true, strips from face-lock
   - Their shots become atmosphere or reaction shots of listeners

9. CHARACTER INTRODUCTIONS
   First appearance should include description in action line:

   Good:   EVELYN RAVENCROFT (early 30s, sharp observant eyes, dark hair,
           practical modern clothing) sits at the kitchen table.

   This gives the story bible and cast system character description data.
   RULE: Age, physical description, and clothing in first appearance
   RULE: Keep description under 100 characters
   RULE: Use visual descriptors (not personality traits)

=== WHAT THE PIPELINE DOES WITH EACH ELEMENT ===

Script Element     → Pipeline Use
------------------------------------------
Scene header       → Creates scene_id, location, time_of_day
                     Used for: location_masters, scene color grades, extras assignment
Character name     → Adds to characters_present, creates cast entry
                     Used for: face-lock references, character_reference_url
Dialogue text      → Stored as dialogue_text on shot
                     Used for: "character speaks:" LTX marker, lip sync direction
Action line        → Stored as beat description
                     Used for: "Character action:" nano injection, "character performs:" LTX
Parenthetical      → Stored as acting direction
                     Used for: emotion layer, performance intensity
Transition         → Creates scene break marker
                     Used for: shot type assignment (SMASH CUT → high intensity)
Atmosphere words   → Classified as mood_cues
                     Used for: scene color grade, lighting direction

=== COMMON MISTAKES THAT CAUSE PIPELINE FAILURES ===

1. MARKDOWN FORMATTING
   Problem: **bold** wraps scene headers, parser can't match regex
   Fix: Remove ALL markdown (bold, italic, headers, dividers)

2. MULTI-LINE DIALOGUE
   Problem: Long dialogue blocks overflow the 250-char LTX limit
   Fix: Break into shorter turns with action beats between

3. MISSING ACTION LINES BETWEEN DIALOGUE
   Problem: Back-to-back dialogue = no visual beats = empty prompts
   Fix: Add at least one action line between every 2-3 dialogue turns

4. GENERIC LOCATIONS
   Problem: "THE ROOM" or "INSIDE" → no location_master match
   Fix: Use specific locations: "RAVENCROFT MANOR LIBRARY", "VILLAGE PUB"

5. UNNAMED CHARACTERS IN ACTION
   Problem: "A man enters" → parser can't track character presence
   Fix: Name characters: "ARTHUR enters through the side door"

6. V.O. CHARACTERS SHOWN ON SCREEN
   Problem: "The Child watches from the doorway" but Child is V.O.
   Fix: V.O. characters are NEVER visible — show listener reactions instead

7. TOO MANY CHARACTERS PER SCENE
   Problem: 6+ characters = face-lock confusion
   Fix: Max 3-4 main characters per scene, extras are unnamed background

=== TEMPLATE ===

TITLE
Episode N: "Episode Title"

INT. LOCATION ONE - TIME

Action line establishing the setting. Specific visual details.

CHARACTER ONE (age, physical description, clothing) does something specific.

CHARACTER ONE
Short punchy dialogue.

Action beat showing physical response or movement.

CHARACTER TWO (age, physical description, clothing) enters or reacts.

CHARACTER TWO
Response dialogue.

CUT TO:

EXT. LOCATION TWO - TIME

New scene with action line establishing new visual context.

(continue pattern)
